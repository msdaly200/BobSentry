## Phase 4 — Triage Report

### Vulnerability Summary

**CVE-2026-45292** affects `io.opentelemetry:opentelemetry-api` ≤ 1.61.x. When the W3C Baggage Propagation header is processed, `W3CBaggagePropagator.extract()` allocates one HashMap entry per `key=value` pair without any limit on entry count or total allocation size.

**Keycloak attack chain (when `--tracing-enabled=true`):**

1. Any incoming HTTP request is intercepted by Quarkus's `InstrumenterVertxTracer.receiveRequest()`
2. This invokes `W3CBaggagePropagator.extract()` on the raw `baggage` header value
3. The `baggage` header is subject only to Vert.x's max-header-size limit (64 KB in Keycloak's config)
4. An attacker sends `baggage: k0=xxxx…,k1=xxxx…` with hundreds of entries filling 60+ KB
5. Each request allocates a large number of String objects; 20 such requests cause JVM heap pressure sufficient to render the server unresponsive

**Unauthenticated attack surface:** The exploit requires only an HTTP request to any Keycloak endpoint — no credentials, no realm, no valid session.

### Remediation

**Primary fix (required):** Upgrade `io.opentelemetry:opentelemetry-api` to **≥ 1.62.0**.

- This requires upgrading Quarkus from `3.33.2.1` → `3.34.x` or applying a BOM override
- Tracking issue: keycloak/keycloak#49549 (referenced in issue comments as the upstream fix vehicle)
- Quarkus LTS 3.33 and 3.27 confirmed as **not receiving this fix** (too risky for LTS)

**Interim mitigations (defence-in-depth):**

1. **Disable tracing if not operationally required:**
   ```
   --tracing-enabled=false
   ```
   This eliminates the attack surface entirely.

2. **Apply a reverse proxy baggage header cap** (e.g. Nginx/Envoy):
   ```nginx
   # Strip oversized or malicious baggage headers at the proxy layer
   proxy_set_header baggage "";
   ```

3. **Reduce max-header-size** in [`quarkus/runtime/src/main/resources/application.properties:64`](quarkus/runtime/src/main/resources/application.properties:64):
   ```properties
   quarkus.http.limits.max-header-size=8192
   ```
   This reduces the maximum baggage payload per request, limiting per-request allocation.
